# Design Pattern Guidelines

Best practices for visual and UI design, prioritized by clarity and consistency. Each rule teaches what to do and why. For exact usage, verify with the design system or tokens.

## Consistency First

Before applying any rule, check existing designs. Many patterns are valid — the best choice is the one consistent with the current project. Inconsistency is worse than a less-than-perfect style.

Check sibling screens, components, or style guides for established patterns. Follow them when they exist.

## Quick Reference
### Claymorphic → `rules/claymorphic.md`
- Soft, layered shapes with subtle shadows. Focus on tactile, 3D-like UI elements.

### Cyberpunk → `rules/cyberpunk.md`
- High-contrast neon colors on dark backgrounds. Futuristic tech aesthetics with glowing elements.

### Editorial → `rules/editorial.md`
- Emphasize typography hierarchy and readability. Large imagery with text overlays.

### Flat → `rules/flat.md`
- Minimal shadows or gradients. Clear geometric shapes. Bold, vibrant color palette.

### Glassmorphic → `rules/glassmorphic.md`
- Frosted glass effect with blur and transparency. Subtle gradients and soft shadows.

### Industrial → `rules/industrial.md`
- Strong, rigid geometric shapes. Metallic textures and muted color palette.

### Japanese-minimalist → `rules/japanese-minimalist.md`
- Clean layouts with ample whitespace. Neutral and soft color palettes.

### Monochrome → `rules/monochrome.md`
- Single-color palette with shades and tints. Focus on contrast and readability.

### Neu-brutalist → `rules/neu-brutalist.md`
- Bold typography with exaggerated spacing. Sharp edges, strong contrasts.

### Neumorphic → `rules/neumorphic.md`
- Soft, extruded UI elements with subtle shadows. Light color backgrounds with gentle highlights.

### Organic-flow → `rules/organic-flow.md`
- Curved, flowing shapes and natural lines. Soft gradients and harmonious colors.

### Pixel-art → `rules/pixel-art.md`
- Grid-based low-resolution imagery. Limited color palette with sharp edges.

### Retro-futurism → `rules/retro-futurism.md`
- Futuristic designs inspired by past visions. Neon gradients, chrome effects, sci-fi typography.

### Typographic → `rules/typographic.md`
- Typography as the primary visual element. Strong hierarchy and alignment.

### Wireframe → `rules/wireframe.md`
- Simplified layouts with placeholder elements. Focus on structure, flow, and hierarchy.


## How to Apply

Always use a sub-agent to read rule files and explore this skill's content.

1. Identify the component or screen type and select relevant sections (e.g., button → §1 Claymorphic, §4 Flat, §10 Neumorphic)
2. Check sibling screens for existing patterns — follow those first per Consistency First
3. Verify design implementation with the design system or token guidelines
